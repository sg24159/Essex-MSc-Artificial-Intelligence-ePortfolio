Agent based systems have a long history that goes back to before modern machine learning. These systems operated on formal rules and logic, handwritten by developers (An, 2026). Is this approach obsoleted by machine learning or is it worth another look, particularly in industries that have high privacy and security requirements?

Rules based agents have a few advantages over neural network based agents. The if-then structure helps with interpretability, which can give a user or supervisor much greater confidence in the agents’ decisions (Sarker et al., 2024). It is easy to update the behaviour of a rules-based agent by adding, removing, or modifying a rule (Sarker et al., 2024). This is in stark contrast to LLM-based agents which can hallucinate and whose text-based configuration may be brittle (An, 2026).

One area where the combined approach has been seeing success is in cybersecurity. The rules based approaches in defensive cybersecurity systems ensure that detected problems are explainable and classified transparently (Sarker et al., 2024). Instead of manually creating rules, which is prohibitively difficult (An, 2026), a data-driven approach can be used instead. In this approach, machine learning systems extract patterns from the data and suggest rules (Sarker et al., 2024). This mitigates the weaknesses of both a purely learning based system and a purely rules based system, but may not be applicable to all agentic use cases.
